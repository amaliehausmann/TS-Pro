Hvad er TypeScript?

TypeScript er et programmeringssprog udviklet af Microsoft, som bygger videre på JavaScript. Typescript er altså stortset bare JavaScript med ekstra features, som kan hjælpe udviklere med at skrive kode der er mere robust og nemmere at vedligeholde.

TypeScript er et superset af JavaScript. Det betyder at al gyldig JavaScript kode også er gyldig TypeScript kode.

TypeScript har nogle ekstra funktioner som JavaScript ikke har, bl.a. static typing og interfaces.

Når man skriver kode i TypeScript, bliver det compiled til helt almindelig JavaScript, som alle browsere kan køre.

Fejlene bliver fanget tidligt i udviklingsprocessen. Når TypeScript bliver compiled bliver der nemlig allerede her tjekket for typefejl og syntaxfejl!

Hvad er formålet med TypeScript?

Formålet med TypeScript, er at forbedre hele JavaScript-udviklingsoplevelsen, bl.a. ved at tilføje static typing og andre funktioner, der gør koden mere robust, nemmere at vedligeholde og nemmere at fejlsøge.

TypeScript forbedre kvaliteten af koden og fejlfindingsprocessen. Her spiller static typing en stor rolle.

TypeScript har en bedre IDE (Integrated Development Enviroment) understøttelse, som bl.a. gør autocompletion med Intellisense markant bedre.

TypeScript er især meget brugbar i store projekter, hvor mange udviklere arbejder sammen.

Hvad er TypeScript's anvendelsesområde?

TypeScript er et meget alsidigt programmeringssprog, som kan anvendes på mange forskellige udviklingsområder.

Frontend: TypeScript anvendes ofte til at udvikle store og kompleske webapplikationer, da det giver en mere strukturet og type-sikker kodebase. Frameworket Angular er bl.a. bygget med TypeScript.

Backend: TypeScript kan bl.a. anvendes til at skrive backend-applikationer i node.js. Nest.js er bl.a. bygget på TypeScript.

Mobilapplikationer: TypeScript kan bl.a. anvendes med React Native i at udvikle mobilapplikationer.

Store Projekter: TypeScript kan altså anvendes til rigtig mange ting, men anvendes primært til store projekter for at sikre god struktur i kodebasen.

Fordele og Ulemper ved TypeScript

Fordele ved TypeScript er bl.a. static typing som øger kvaliteten af koden, og mindsker mængden af bugs. TypeScripts IDE Support giver også en bl.a. bedre autocompletion.

Andre fordele ved TypeScript er at det understøtter mange forskellige libraries og frameworks, og at det kan inkorporeres i koden gradvist.

Nogle af ulemperne er bl.a. at det er mere komplekst end almindeligt JavaScript, og at det kræver at blive compiled for at kunne anvendes.

Udover det kan det være mere besværligt at anvende TypeScript til små projekter, og TypeScript kan være meget restriktivt.
